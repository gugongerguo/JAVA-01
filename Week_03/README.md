***1.对高性能的理解：***

    1.1高并发用户 系统表现出来的支持更多用户同时使用
    1.2高吞吐量 QPS每秒请求量
    1.3低延迟  高频交易系统 
    1.4以上整体称作高并发

***2.压测 wrk -c 40 -d 30s --latency http://localhost:3000***

    2.1 -c 并发量
    2.2 通过分析延迟p50 p90 p99，来查看延迟的分布，单单只看平均延迟不太准确
    2.3 一般延迟越低吞吐越高，对于多个系统来回调用就不能只根据延迟来判断吞吐量
    注意：延迟和响应时间的差别，延迟是系统内部，响应时间还要加上网络通信时间

***3.高性能副作用***

    3.1 复杂度高
    3.2 建设维护成本高
    3.3 一旦出现bug后果很严重
